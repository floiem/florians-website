Ambient Chat

Not just another chatroom

The first idea behind Ambient Chat was to create a basic chat room with a nice little UI coded with VueJS on the Frontend and a NodeJS Express server on the backend with SocketIO in between. The idea was soon expanded due to the easiness of the implementation with todays technology. The final version is a full chat application which is receiving, saving and sending out messages to other people that are logged in. In addition, you could talk to Artificial Intelligence with the help of OpenAI and GPT-3 to not only write with humans but also with realistic bots.

Process

Blueprinting what's important

After sketching a little bit and thinking about how I want to architecture my. application I started with creating classes and components using the Frontend Framework VueJS.
Thereafter I started creating the first component which I knew I would need later in the process. I did the same on the Backend, after initialising the NodeJS server and installing the Express framework I made first tests for sending and receiving data.

Connecting with Websockets

For communicating between Frontend and Backend I had to use a technology. The goals could be reached with HTTP requests, another communication protocol or - like I did - with websockets or to be more clear: socket.io. SocketIO helped me creating a bidirectional communication between server and client.

Styling the interface

I styled the interface with CSS which was a lot of fun. I didn't want the interface to look like WhatsApp, Facebook Messenger or Telegram. So I created a different layout with using and styling different components that later resulted in an interface that is easy to understand and easy to use.

Using advanced technology

After I was finished with the basic project I had the opportunity to start with something advanced. Therefore I have implemented some more complex technology into my application such as Artificial Intelligence with the help of GPT-3, a easy database with Airtable and the possibility to upload files (images) as avatars which are saved into the filesystem of the server.
